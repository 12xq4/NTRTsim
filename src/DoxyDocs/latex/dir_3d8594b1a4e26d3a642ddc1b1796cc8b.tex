\subsection{Detailed Description}
Classes used to build a tensegrity out of rigid objects and strings. Uses a specification of \hyperlink{classtg_nodes}{tg\-Nodes} and \hyperlink{classtg_pairs}{tg\-Pairs} to build a tensegrity structure. Nodes are first placed into a \hyperlink{classtg_structure}{tg\-Structure}, and then those nodes are paired and tagged. A \hyperlink{classtg_pair}{tg\-Pair} can either become a \hyperlink{classtg_rigid_info}{tg\-Rigid\-Info} or a \hyperlink{classtg_connector_info}{tg\-Connector\-Info} based on its tg\-Tag structure. This is determined by the config (from \hyperlink{classtg_rod_info}{tg\-Rod\-Info} or \hyperlink{classtg_linear_string_info}{tg\-Linear\-String\-Info}) that is passed into a \hyperlink{classtg_build_spec}{tg\-Build\-Spec}. The \hyperlink{classtg_build_spec}{tg\-Build\-Spec} is given to a \hyperlink{classtg_structure_info}{tg\-Structure\-Info}, which then builds the structure into the relevant \hyperlink{classtg_model}{tg\-Model}. It takes care of compouding \hyperlink{classtg_rod}{tg\-Rod} (s) that share the same nodes using \hyperlink{classtg_rigid_auto_compound}{tg\-Rigid\-Auto\-Compound}.

For an example, see \hyperlink{class_prism_model}{Prism\-Model}
